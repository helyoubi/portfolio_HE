# Agent : Internationalisation (FR/EN)

**À consulter pour :** ajouter/modifier une traduction, créer une chaîne multilingue, debug du switch de langue.

## Architecture

| Élément | Implémentation |
|---|---|
| Gestionnaire | `scripts/languageManager.js` |
| Persistance | `localStorage['language']` (`'fr'` ou `'en'`) |
| Défaut | `'fr'` |
| Contenu données | `data/portfolioData.json` (clés `en`/`fr`) |
| Navigation/menu | `scripts/hamburgerMenu.js → updateNavigationLanguage()` |

## Règles invariantes

1. **Chaque chaîne UI a une version FR et EN** — jamais de fallback silencieux.
2. **Les tableaux de données ont la même longueur dans `en` et `fr`** (1 entrée FR = 1 entrée EN).
3. **Switch sans rechargement de page** — re-rendu dynamique uniquement.
4. **Format des dates par langue** : "Juillet 04, 2025" (FR) vs "July 04, 2025" (EN).

## Ajouter une nouvelle chaîne

### Cas A : contenu dynamique (cartes, sections)
Ajouter dans `portfolioData.json` sous `en` et `fr`. Le rendu se fait via `dataLoader.js`.

### Cas B : libellé statique (boutons, menus, titres de section HTML)
1. Ajouter la traduction dans `updateNavigationLanguage()` ou la fonction équivalente
2. Marquer l'élément cible avec un attribut data ou un id stable
3. Le listener `languageChanged` met à jour le texte

## Tester le switch

- [ ] Bascule FR → EN : tout le texte change, aucune chaîne ne reste figée
- [ ] Bascule EN → FR : idem
- [ ] Refresh : la langue persiste
- [ ] Pas d'entrée orpheline (présente dans une langue, absente dans l'autre)

## Anti-patterns

- Hardcoder une chaîne dans le HTML sans mécanisme de traduction
- Utiliser le navigateur (`navigator.language`) pour deviner — toujours respecter le choix utilisateur stocké
- Concatener des chaînes ("Updated " + date) → préférer des templates complets par langue
